## Recurring UI Breakage Notes

The repeated failure pattern has been card/list layouts that visually depend on a fixed or underestimated height while the content row contains wrapped chips/buttons.

Observed root causes:

- Dense action chips were allowed to overflow visible card bounds.
- Grid rows did not always expand around wrapped action rows.
- Generic button sizing rules overrode compact sidebar/card-specific rules.
- Mobile cannot rely on hover states to distinguish chips from buttons, so visual affordance needs to be explicit.

Required QA checks for future UI changes:

- Test both desktop and mobile widths before shipping any panel/card/list change.
- For repeated cards, verify adjacent sibling rectangles do not overlap.
- Check that source chips and action buttons remain inside the card after labels wrap.
- Verify the same component after state changes, not only on initial render.
- Avoid `overflow: visible` on dense cards unless the parent layout explicitly reserves space.
- Keep action rows in normal layout flow, preferably flex column/row wrapping with bounded labels.
- Treat Service Worker and Worker cache versions as part of UI QA when stale fallback data affects what the user sees.
